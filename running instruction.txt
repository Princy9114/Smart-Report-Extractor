Here is a step-by-step guide to running the Smart Report Extractor project from scratch:

### 1. Navigate to the project directory
Open your terminal (Command Prompt, PowerShell, or bash) and point it to the project folder where the [main.py](cci:7://file:///e:/SRE/smart-report-extractor/main.py:0:0-0:0) is located.
```bash
cd e:\SRE\smart-report-extractor
```

### 2. Create a Virtual Environment (Recommended)
It's best practice to install dependencies in an isolated virtual environment so it doesn't conflict with your global Python installation.
```bash
python -m venv .venv
```
Activate the virtual environment:
- **Windows (PowerShell):** `.\.venv\Scripts\Activate.ps1`
- **Windows (Command Prompt):** `.\.venv\Scripts\activate.bat`

### 3. Install Dependencies
Install all required packages from [requirements.txt](cci:7://file:///e:/SRE/smart-report-extractor/requirements.txt:0:0-0:0).
```bash
pip install -r requirements.txt
```

### 4. Download the spaCy NLP Model
Layer 2 relies on the `en_core_web_sm` Language model to extract entities like names and dates. You must download it after installing `spacy`.
```bash
python -m spacy download en_core_web_sm
```

### 5. Set up Environment Variables
Create a file named `.env` in the root of `smart-report-extractor/` to hold your API keys. Copy the following into it and insert your real API key if you want Layer 4 (the LLM fallback) to work:
```ini
# Add your Anthropic key here so the LLM fallback layer can activate on low confidence text:
ANTHROPIC_API_KEY=sk-ant-api03-...

# Optional configurations (these defaults are safe to leave out)
ANTHROPIC_MODEL=claude-3-5-haiku-20241022
ANTHROPIC_MAX_TOKENS=2048
```
*(Note: If you don't enter an API key, the pipeline perfectly handles offline extractions via pdfplumber, regex, and spaCy, but skips step 5 fallback).*

### 6. Run the Server
Start the FastAPI server using `uvicorn`. The `--reload` flag automatically refreshes the server if you alter the python files.
```bash
uvicorn main:app --reload
```

### 7. Open the Frontend
Once running, the application will tell you it's listening on `http://127.0.0.1:8000`.

Open your web browser and navigate to:
- **Web UI:** [http://127.0.0.1:8000](http://127.0.0.1:8000) *(The drag & drop front end)*
- **API Docs:** [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs) *(Interactive Swagger API sandbox)*
